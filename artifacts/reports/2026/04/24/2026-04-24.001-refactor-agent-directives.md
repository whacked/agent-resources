# Refactor Agent Directives into Workflow Guides

Keep `AGENTS.md` short and move detailed report and cumulative-data procedures into focused agent guides.

# Context

As long-running projects accumulate procedures, a single large `AGENTS.md` risks burying important triggers and operational details. Agents need a small always-visible routing layer plus task-specific documents that are read when a workflow is activated.

# Analysis

Reports and cumulative CPD data have different operational details, validation commands, and evolution policies. Keeping those details inline in `AGENTS.md` makes the root directive longer and less effective over time. A short root file can remain salient while mandatory workflow links preserve exact procedures.

# Decision

Refactor `AGENTS.md` into a routing layer with workflow triggers and invariants. Move report-writing instructions to `docs/agent-guides/reports.md` and cumulative CPD data instructions to `docs/agent-guides/cpd-data.md`.

The root directive now tells agents to read the relevant guide before writing matching artifacts. The CPD guide also introduces optional `DATASET.md` manifests for long-lived accumulations.

# Results

- `AGENTS.md` stays short enough to remain high-signal in agent context.
- Detailed workflow rules remain available in focused documents.
- Report and CPD data workflows retain their validation and normative-report requirements.
