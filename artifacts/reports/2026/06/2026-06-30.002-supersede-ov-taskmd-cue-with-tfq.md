# Supersede ov, taskmd, and cue with the single tfq binary

Replace the three external CLI dependencies `ov` (Obsidian vault navigation),
`taskmd` (markdown task tracking), and `cue` (frontmatter validation) with a
single binary, `tfq` (text-file-query), across every skill, script, schema-
validation path, dependency manifest, and dev-shell in agent-resources. `ck`
(semantic search) is explicitly retained — tfq does not do embeddings.

# Motivation

The skills shelled out to three separate binaries for what is fundamentally one
problem: reading, writing, and validating a directory of frontmatter'd markdown
records. That carried real cost:

- **Install surface.** Three tools to provision (`pkgs.cue`, a prebuilt `ov`
  fetchzip, a prebuilt `taskmd` fetchzip), each from a different upstream.
- **An index to babysit.** `ov` required `ov index build` after every write
  session before `ov search`/`ov backlinks` would see new content — a step the
  notes and synthesize loops had to remember and re-run per vault.
- **A CWD footgun.** `taskmd` resolved its config by walking up from the current
  directory, so `new-task.sh` had to `cd` into `agents/tasks/` and `mv` the
  created file into the shard, and a stray `cd` could poison the tool's working
  directory for later calls.
- **Reality check.** On the working machine none of `ov`, `taskmd`, or `cue`
  were even on `PATH` (`command -v` found none) — only `ck`, `rg`, `jq`. The
  skills referenced binaries that were not installed.

tfq was authored specifically to subsume these three (see its
`docs/agent-guides/superseding-ov-taskmd-cue.md`): one Go binary, index-free,
RE2/ripgrep search, with the cuelang library bundled for `cue vet`-equivalent
validation and no `cue` binary required.

# Analysis

Functional fidelity was verified against the repo's real schemas and live
sample data before any file was rewritten:

- **cue → `tfq --validate FILE --schema TPL`.** Valid fixtures pass (exit 0);
  invalid fixtures fail (exit 1) with per-field CUE errors, including RE2 `=~`
  constraints and the unquoted-YAML-date normalization the old `cue vet` relied
  on. tfq reads the ```` ```cue ```` block directly from the markdown template,
  so the `extract_cue_schema` awk is no longer needed. The repo's existing
  `scripts/tests/run-tests.sh` passes 3/3 unchanged.
- **taskmd → `tfq --task/--list/--next/--set/--done/--graph`.** Same one-file-
  per-record model: sharded `YYYY/MM/NNN-slug.md`, padded sequential ids,
  taskmd-compatible frontmatter. `--depends-on` writes a real YAML list that
  gates `--next`; marking a blocker `--done` unblocks its dependents; `--graph`
  emits the dependency edges.
- **ov → `tfq <search>/--show/--links/--backlinks/--tags/--list`.** Index-free,
  structural `--in heading|tag|link` narrowing, `--json` for tooling. Backlinks
  are computed live per query.

Two known scope boundaries, both acceptable: `ov append --section` (structural
in-place insert) has no tfq equivalent, but the notes loop creates-and-supersedes
rather than appends; and `ov index build` also fed Obsidian's own backlink panel
— Obsidian now maintains that index itself, which is fine for a human-browsed
vault.

The dev-shell tfq package was built and verified through nix
(`buildGoModule`, `vendorHash` pinned, `nativeCheckInputs = [ ripgrep ]` so the
search tests find `rg` at check time); the resulting binary reports its version
and validates correctly.

# Decision

tfq is now the sole tool for notes, tasks, and frontmatter validation in this
repo. Concretely:

1. **New canonical skill** `skills/tfq/SKILL.md` documents the unified verb set
   (search/show/links/tags, task lifecycle + dependencies, validate). Its
   description routes the model to tfq for any operation the three old tools
   covered.
2. **`skills/ov/` and `skills/taskmd/` are deleted.** Their verbs live in the
   tfq and notes skills.
3. **Scripts rewired:** `validate-frontmatter.sh` calls `tfq --validate` instead
   of `cue vet`; `new-task.sh` is one `tfq --root agents/tasks --task` call
   (csv `--tags` → repeated `--tag`, `--context FILE` → `--field context=`);
   `validate-note.sh` guards on `tfq` instead of `cue`.
4. **doctor** requires `tfq` + `rg` + `jq`, treats `ck`/`cpd` as optional, drops
   the `ov index status` and `.taskmd.yaml` checks, and registers the new skill
   set (`tfq ck audit-skills doctor notes synthesize`).
5. **bootstrap** drops `taskmd init` (tfq is index-free, no per-collection
   config) and instead confirms `tfq` is on PATH.
6. **shell.nix** drops `pkgs.cue`, `obsidianVault`, and `taskmd`; adds a pinned
   `buildGoModule` tfq derivation. **README** and the `notes`/`synthesize`/`ck`
   skills are updated to the tfq verbs.

`tfq` becomes a **required** dependency on `PATH` (built from
github.com/whacked/tfq via `make build`, or via the dev-shell). `rg` and `jq`
remain required; `ck` and `cpd` remain optional.

# Consequences

- One binary to install instead of three; no search index to build or keep
  fresh; no per-CWD task config to resolve. Collections are addressed explicitly
  with `--root agents/tasks` / `--root agents/notes`.
- The `scripts/validate-frontmatter.sh` CLI signature is unchanged
  (`<template> <document>`), so `docs/agent-guides/reports.md` and the report
  validation workflow keep working as-is — only the internals swapped.
- Verified end-to-end in a controlled vendored workspace: `run-tests.sh` 3/3;
  `new-task.sh` produces sharded, dependency-gated tasks; `validate-note.sh`
  PASSes valid notes and FAILs invalid frontmatter with tfq's CUE errors; and
  `doctor` reports all binaries, all six skills, and tfq-backed frontmatter
  validation green.
- Historical design records under `docs/superpowers/specs|plans/` are left
  intact as dated artifacts; only living docs and code were migrated.
- Follow-ups if a real need appears: add `ov append --section` semantics to tfq;
  and bump the pinned tfq rev/hashes in `shell.nix` when upstream advances.
